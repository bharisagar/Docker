# Docker Project Guide

This guide covers fundamental Docker concepts and commands across a four-day learning journey.

## Day 1: Docker Basics and Installation

### Detailed Topics Covered
On Day 1, we delved into the foundational aspects of Docker. We started with an **Introduction to Docker**, understanding its role as a platform for developing, shipping, and running applications in containers. The core benefit of Docker—encapsulating an application and its dependencies into a standardized unit for software development—was a key takeaway. Following this, we covered the **Docker installation and setup** process specific to your operating system, ensuring all necessary components were correctly installed and configured. Finally, we learned to **verify Docker installation** by running a simple command, confirming that Docker was operational on the system.

### Docker Commands Used
```bash
docker --version
docker run hello-world
```

## Day 2: Docker Images and Containers

### Detailed Topics Covered
Day 2 focused on the two core components of Docker: **images** and **containers**. We started by **understanding Docker images** as lightweight, standalone, executable packages that include everything needed to run a piece of software, including the code, a runtime, libraries, environment variables, and config files. We then moved to **managing Docker containers**, which are runnable instances of an image. We practiced **running interactive containers**, allowing us to directly interact with the operating system inside a container. We also learned how to **list and inspect images and containers** to view their properties and states. Finally, we covered the essential operations of **stopping and removing containers and images** to manage system resources efficiently.

### Docker Commands Used
```bash
docker pull <image_name>
docker images
docker run -it ubuntu bash
docker ps
docker ps -a
docker stop <container_id>
docker rm <container_id>
docker rmi <image_id>
```

## Day 3: Dockerfile and Building Custom Images (Multi-stage)

### Detailed Topics Covered
Day 3 was all about customizing and optimizing Docker images. We started with **Dockerfile syntax and best practices**, learning how to write instructions to assemble an image. This included commands like `FROM`, `RUN`, `COPY`, and `EXPOSE`. We then applied this knowledge to **building custom Docker images** from our Dockerfiles. A crucial part of this day was **understanding image layers with `docker history`**, which showed us how each instruction in a Dockerfile creates a new, read-only layer, impacting image size and build efficiency. We concluded with an **introduction to multi-stage builds for optimized images**, a powerful technique to create smaller, more secure images by separating build-time dependencies from runtime dependencies.

### Docker Commands Used
```bash
docker build -t <image_name> .
docker history <image_name>
docker inspect <image_name>
```

## Day 4: Docker Volumes and Networking

### Detailed Topics Covered
On our final day, Day 4, we explored advanced Docker features for data management and inter-container communication. We began with **managing data persistence with Docker volumes**, understanding how volumes provide a way to store data generated by and used by Docker containers, ensuring data is not lost when a container is stopped or removed. This contrasted with the ephemeral nature of container filesystems. Next, we moved to **understanding Docker networking concepts**, learning how containers communicate with each other and the outside world. We practiced **creating and managing custom networks** to isolate container traffic and enable name-based resolution. Finally, we learned about **connecting containers using networks**, demonstrating how to link multiple containers within a custom network to build multi-service applications.

### Docker Commands Used
```bash
docker volume create myvolume
docker volume ls
docker run -d --name devtest --mount source=myvolume,target=/app nginx
docker inspect devtest
docker network create mynetwork
docker network ls
docker run -d --name mynginx --network mynetwork nginx
docker run -d --name mynode --network mynetwork mynodeimage
docker network inspect mynetwork
```
