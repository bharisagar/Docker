Real-World Problem

Consider a PostgreSQL database running inside a container.

Without a Docker Volume:

Container Deleted
        ↓
Database Deleted
        ↓
Data Lost

This is not acceptable in production.

To solve this problem, Docker provides Volumes.

Volumes store data outside the container lifecycle.

Even if the container is removed, the data remains safe.

What is a Docker Volume?

A Docker Volume is a storage mechanism used to persist data generated by containers.

Without Volumes:

Container Deleted
       ↓
Data Lost

With Volumes:

Container Deleted
       ↓
Data Still Exists

Volumes are commonly used with:

PostgreSQL
MySQL
MongoDB
Redis
Elasticsearch

Running PostgreSQL Container

Command:

docker run -d \
--name postgres-db \
--network day4-network \
-e POSTGRES_USER=admin \
-e POSTGRES_PASSWORD=admin123 \
-e POSTGRES_DB=studentdb \
-v postgres-data:/var/lib/postgresql/data \
-v /c/bari_sagar/Docker/day4/sql/init.sql:/docker-entrypoint-initdb.d/init.sql \
-p 5432:5432 \
postgres:15

Explanation:

Parameter	Purpose
--name	Container Name
--network	Connect to Docker Network
-e	Environment Variables
-v	Attach Docker Volume
-p	Port Mapping
Evidence - PostgreSQL Container Running

Verification:

docker ps

The PostgreSQL container was successfully started and exposed on port:

5432
Connecting to PostgreSQL Using Docker Network

Instead of connecting from the host machine, we launched another PostgreSQL client container.

Command:

docker run -it --rm \
--network day4-network \
postgres:15 \
psql -h postgres-db -U admin -d studentdb

Password:

admin123

Important:

Notice the hostname:

postgres-db

This is the container name.

Docker DNS automatically resolves:

postgres-db

to the actual container IP address.

No manual IP configuration is required.

What is Docker DNS?

When containers are connected to the same network:

Container A
      |
      |
      V
Container B

Container A can access Container B using:

container-name

instead of:

172.x.x.x

Example:

postgres-db

This feature is called Docker DNS Service Discovery.

What is .dockerignore?

The .dockerignore file is used to exclude files and directories from the Docker build context.

Example:

.git
*.log
node_modules
.env
__pycache__

Benefits:

Smaller images
Faster builds
Better security
Reduced build time

Interview Questions
What is a Docker Volume?

A Docker Volume is a persistent storage mechanism used to store container data outside the container lifecycle.

Why do we need Docker Volumes?

Volumes ensure data is not lost when containers are stopped, deleted, or recreated.

What is a Docker Network?

A Docker Network enables communication between containers.

How do containers communicate?

Containers communicate using container names through Docker DNS.

Example:

postgres-db

instead of IP addresses.

What is Docker DNS?

Docker DNS automatically resolves container names into IP addresses.

What is the difference between Bridge and Host Network?

Bridge:

Default Docker network
Containers communicate through Docker networking

Host:

Shares host networking stack
No network isolation
What is .dockerignore?

A file used to exclude unnecessary files from Docker build context.
